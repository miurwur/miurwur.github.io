# Лабораторная работа №3  
## Цель работы  
Настроить процесс CI/CD для статического сайта. Добавить репозиторий на SourceCraft  

## Ход Работы  
1. Создан пустой репозиторий для проекта на SourceCraft  
2. Сгенерирован персональный токен доступа  
3. Добавление удаленного репозитория   
```python
git remote add sourcecraft  
```
4. Проверка списка удаленных репозиториев  
```python
git remote -v
```
<img src="/img/remove-v.PNG" alt="remove-v" width="200"> 

5. Настройка CI/CD (Для автоматизации процесса были созданы конфигурационные файлы в директории .sourcecraft/ и .github/)  
Описанный workflow автоматически запускается при каждом push в ветку main.   
Основные этапы(cubes):  
- Сборка: использование python:3.14-slim, установка зависимостей из requirements.txt и генерация статических HTML-файлов командой mkdocs build  
- Публикация: Создание ветки release, добавление собранного сайта из папки ./site, отправка изменений в удаленный репозиторий.  

Разница в CI/CD для GitHub и SourceCraft:  
GitHub:  
- используется mkdocs gh-deploy - черный ящик, сам автоматически создает ветку gh-pages, собирает туда сайт и делает push  
SourceCraft:  
- Настройка вручную через кубы. Прописываем шаги(создание ветки release, добавление папки site, коммит и push). Отдельный файл sites.yaml связывает ветку с веб-хостингом  

6. Развертывание сайта
После выполнения команды git push sourcecraft main на SourceCraft успешно собран сайт. Итоговые файлы сайта размещены в ветке release репозитория.
