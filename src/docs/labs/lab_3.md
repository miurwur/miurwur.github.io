## Настройка CI/CD 
- Для автоматизации процесса были созданы конфигурационные файлы в директории .sourcecraft/ и .github/
Описанный workflow автоматически запускается при каждом push в ветку main. 
- Cubes - Сборка: использование python:3.14-slim, установка зависимостей из requirements.txt и генерация статических HTML-файлов командой mkdocs build  
- Cubes - Публикация: Создание ветки release, добавление собранного сайта из папки ./site, отправка изменений в удаленный репозиторий.  

Разница в CI/CD для GitHub и SourceCraft:  
GitHub:  
- используется mkdocs gh-deploy - команда сама автоматически создает временную папку, инициализирует там git, копирует собранный сайт, фиксирует изменения в ветку gh-pages и делает push  
- используются готовые экшены(actions), который уже специализированы под GitHub
SourceCraft:  
- Настройка вручную через кубы. Прописываем шаги(создание ветки release, добавление папки site, коммит и push). Отдельный файл sites.yaml связывает ветку с веб-хостингом  

### Развертывание сайта
После выполнения команды git push sourcecraft main на SourceCraft успешно собран сайт. Итоговые файлы сайта размещены в ветке release репозитория.
